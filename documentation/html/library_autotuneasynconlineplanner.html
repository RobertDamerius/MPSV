AutoTune AsyncOnlinePlanner

Library

MPSV / AutoTune AsyncOnlinePlanner

Description

Calculate all tuning parameters for the MPSV AsyncOnlinePlanner based on given vehicle parameters and one intuitive tuning parameter.

Input Ports

timestamp

The timestamp to be set for the parameter data whenever at least one parameter changes.

controlEffort

The amount of control effort in range (0,1] used to drive the vehicle in a steady-state in longitudinal direction. A low value makes the vehicle move slower and a high value makes the vehicle move faster.

geometry_verticesVehicleShape

A 2-by-100 matrix of type double containing a vertex list for multiple convex polygons, where each polygon is separated by non-finite values. All non-finite values of the matrix are ignored. Example for a simple rectangle of 10-by-2 meters: [5 -5 -5 5 nan(1,96); 1 1 -1 -1 nan(1,96)]

geometry_collisionCheckMaxPositionDeviation

Maximum position deviation for path subdivision during collision checking. Must be at least 0.01 meters.

geometry_collisionCheckMaxAngleDeviation

Maximum angle deviation for path subdivision during collision checking. Must be at least 1 degree (0.0174532925199433 radians).

model_matF

3-by-12 coefficient matrix (row-major order) of model nu_dot = F*n(nu) + B*tau.

model_matB

3-by-3 input matrix B (row-major order) of model nu_dot = F*n(nu) + B*tau.

model_vecSatXYN

Absolute saturation value for input vector.

model_vecTimeconstantsXYN

Timeconstants {TX, TY, TN} for input force dynamics.

model_vecTimeconstantsInput

Timeconstants {Tf1, Tf2, Tf3} for input filter dynamics.

timing_sampletime

Sampletime to be used for fixed-step trajectory simulation.

timing_maxComputationTimePath

Maximum computation time in seconds for path planning.

timing_maxComputationTimeMotion

Maximum computation time in seconds for motion planning.

metric_weightPsi

Weighting (greater than 0) for heading angle (psi) in distance metric function.

metric_weightSway

Weighting (greater than or equal to 0) for sway movement (heading angle with respect to perpenticular direction of movement).

metric_weightReverseScale

Weighting (greater than or equal to 0) for sway and reverse movement (heading angle with respect to line angle).

metric_weightReverseDecay

Decay factor (greater than 0) for the weighting function that weights sway and reverse movement.

metric_distanceScale

Scale factor (greater than or equal to 0) for obstacle distance d cost in cost map: scale*exp(-decay*d^2).

metric_distanceDecay

Decay factor (greater than 0) for obstacle distance d cost in cost map: scale*exp(-decay*d^2).

asyncOnlinePlannerParameter

A bus, which defines the parameter for the asynchronous online planner. This output is equal to that of the Default AsyncOnlinePlanner Parameter Bus block.

Details

This section provides details about the empirical tuning.

timings

The computation time on reset is set to 20 percent of the maximum computation times. The additional ahead planning time is max(0.2, 10*sampletime).

skeletalPoints

The skeletal points are two points [xmax; ymean] and [xmin; ymean], where xmax and xmin are the maximum and minimum vertex coordinates of the vehicle shape vertices, respectively, and ymean is the average y coordinate of the vehicle shape vertices.

costMapResolution, costMapModBreakpoints

The modulo factor for breakpoints is set to 10 and the resolution is 1 percent of the largest vehicle dimension.

matK, maxRadiusX, maxRadiusY, maxRadiusPsi

The controller gain and guidance radii are connected to each other. First, the longitudinal stop distance is calculated based on the maximum force according to the saturation value of the model. The desired radius in x direction is then set to 1.2 * stopDistance. Afterwards, the controller gain is adjusted in a bisection search until the control effort would match the specified controlEffort value. The guidance radii are calculated based on the controller gain matrix and the steady-state force and velocity.

minRadiusPosition

This value is the lowest value of maxRadiusX, maxRadiusY as well as the minimum vehicle dimension.

rangePose, rangeUVR, rangeXYN

The range is calculated based on vehicle dimension, maximum velocity and maximum force, e.g. 5 percent of the maximum value. For sway speed, 10 percent is used. The angular range is fixed to 5 degrees.

samplingRangePosition, samplingRangeAngle

The angular sampling range around a path is fixed to 30 degrees. The positional range is calculated based on the guidance radius.

maxInputPathLength

This length is calculated from the maximum velocity and total replanning time and the greatest guidance radius such that a subgoal pose is not within the guidance radius within the total replanning time.

maxUVR (information-purpose only)

This value represents the maximum velocity that would be reached when driving with a maximum force according to the model saturation value. Each DoF is considered separately, assuming that matB of the model is mainly diagonal.

steadyStateUVR (information-purpose only)

This value represents the steady-state velocity that is reached when driving with a nominal force of controlEffort*satXYN, where satXYN is the input saturation value of the model. Each DoF is considered separately, assuming that matB of the model is mainly diagonal.
